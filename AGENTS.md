# 工作规则

## 基准需求文档

- 原始软件需求规格说明书位于 `docs/a股板块主线雷达_软件需求规格说明书_v_1.md`。
- 该文件是需求基准文件，后续工作不得修改、格式化、重命名或自动整理该文件。
- 如需引用 SRS 内容，只能在其他文档中摘要说明，不直接改动原文件。

## 决策记录

- 后续产品决策、技术方案或实现取舍如果与 SRS v1.0 不一致，必须记录到 `docs/DESIGN.md` 的“与 SRS 的差异决策记录”章节。
- 差异记录需包含：日期、决策、与 SRS 的差异点、原因、影响范围。
- `MEMORY.md` 用于记录当前有效决策和长期约束；`TODO.md` 用于记录可执行任务。

## 提交流程

- 每完成一个明确步骤，执行一次 Git 提交。
- 不提交运行产物、缓存、日志、编译生成的 exe 或本地导出的行情数据。
