## 2. 背景与目标

### 2.1 背景

A股市场板块轮动速度快，热点经常在多个行业、概念和题材之间切换。传统的板块热度榜通常只关注单一指标，例如涨幅、成交额、资金流入、涨停数量或舆情热度。这类方法存在明显缺陷：

1. 容易把“已经高潮、次日容易分歧”的板块排到前列；
2. 容易被单日大涨、单日涨停潮或舆情刷屏误导；
3. 无法区分“当日最热”和“未来1—3个交易日更可能延续”；
4. 无法识别资金接力断裂、核心股走弱、后排不跟等退潮信号；
5. 板块定义碎片化，标准概念板块与市场真实主线存在偏差；
6. 缺少模型置信度表达，容易给出看似精确但实际可靠性较低的排序。

因此，需要建设一套面向A股短线与中短线复盘的“板块主线雷达”系统，综合成交、涨停、板块广度、核心股结构、价格强度、舆情、催化、风险、板块聚类和置信度等信息，辅助用户识别当前市场主线和潜在延续方向。

### 2.2 产品目标

本系统目标是构建一个可日常复盘、盘中监控、历史回测和策略迭代的板块主线识别平台，帮助用户回答以下问题：

1. 今天市场最热的板块/主线是什么？
2. 哪些板块只是当日情绪爆发，哪些更可能延续？
3. 当前主线处于启动、加速、高潮、分歧、退潮还是修复阶段？
4. 哪些板块存在资金接力断裂、炸板、核心股走弱、舆情过热等风险？
5. 底层概念如何聚合为更符合交易语境的上层主线？
6. 模型当天的判断置信度高、中、低分别是什么原因？
7. 哪些因子近期有效，哪些因子需要降低参考权重？
8. 历史上该模型对次日、3日、5日板块延续性的预测效果如何？

### 2.3 设计原则

1. **权重少动，风险扣分多动**：基础因子权重保持稳定，避免因主观市场环境判断带来误差；风险扣分根据客观负面信号动态变化。
2. **热度与延续性分离**：当日热度用于识别市场注意力，延续性用于评估未来1—3个交易日继续走强的概率。
3. **舆情看边际，过热要扣分**：舆情绝对热度不宜高权重，舆情变化率更适合捕捉从冷到热的早期变化。
4. **底层板块计算，上层主线输出**：用标准行业/概念板块计算数据，用市场主线语言输出结果。
5. **核心股优先于后排噪声**：核心股、容量中军和连板龙头的状态对板块延续性具有更高解释力。
6. **输出置信度**：模型不能假设每天都能看清市场，应在低置信度环境下明确提示结果参考价值有限。
7. **可回测、可解释、可迭代**：每个评分、扣分和结论都应能追溯到数据与规则。

## 3. 系统范围

### 3.1 范围内功能

系统需支持以下能力：

1. 数据采集与清洗；
2. 底层行业/概念板块评分；
3. 上层市场主线聚合；
4. 热度分、延续性分、风险扣分、主线分计算；
5. 舆情绝对热度与舆情边际变化计算；
6. 资金接力断裂识别；
7. 动态风险扣分；
8. 因子近期有效性评估；
9. 模型置信度计算；
10. 板块主线榜单输出；
11. 单个主线详情分析；
12. 盘中监控与收盘复盘；
13. 历史回测与模型评估；
14. 参数配置与版本管理；
15. 异常数据监控与日志审计；
16. 前端可视化看板；
17. API服务输出。

### 3.2 范围外功能

第一阶段暂不包含以下能力：

1. 自动交易下单；
2. 个股买卖点推荐；
3. 组合调仓建议；
4. 个股基本面深度估值；
5. 高频逐笔交易策略；
6. 面向公众的投资建议生成；
7. 无人工审核的自动新闻解读交易信号。

### 3.3 目标用户

| 用户类型 | 主要诉求 |
|---|---|
| 短线交易者 | 识别当日最强题材、涨停扩散和次日接力风险 |
| 中短线交易者 | 识别3—7日有延续性的主线方向 |
| 量化研究员 | 回测板块因子有效性、优化评分模型 |
| 复盘分析师 | 生成日度市场主线复盘报告 |
| 产品/投研团队 | 监控市场资金偏好、热点迁移与题材演化 |

## 4. 术语定义

| 术语 | 定义 |
|---|---|
| 底层板块 | 数据源提供的标准行业、概念、主题或指数板块，例如半导体、CPO、稀土永磁、工业气体 |
| 上层主线 | 由多个相关底层板块聚合形成的市场交易主线，例如AI硬件、资源涨价、新能源反弹 |
| 热度分 | 衡量板块当日市场关注度和资金进攻强度的分数 |
| 延续性分 | 衡量板块未来1—3个交易日继续走强可能性的分数 |
| 风险扣分 | 对板块过热、分歧、接力断裂、核心股走弱等负面信号的惩罚分 |
| 主线分 | 综合热度、延续性和风险后的最终排序分 |
| 置信度 | 模型对当日排序可靠性的高/中/低判断 |
| 核心股 | 对某条主线具有代表性和定价影响力的龙头股、中军股或高辨识度个股 |
| 中军股 | 板块内成交额大、市值较高、能承接大资金的核心个股 |
| 龙头股 | 板块内最具短线辨识度、涨幅或连板高度领先的个股 |
| 资金接力断裂 | 板块每天上涨个股频繁更换、昨日领涨股不延续、核心股走弱而后排补涨的现象 |
| 舆情边际变化 | 当前舆情热度相对过去若干日均值的增幅 |
| 舆情绝对热度 | 当前时点某板块的讨论量、搜索量、新闻提及量或社区热度绝对水平 |
| 炸板率 | 触及涨停但未能封住的股票数量占触板股票数量的比例 |
| 封板率 | 最终涨停股票数量占触板股票数量的比例 |

## 5. 总体业务流程

### 5.1 日度收盘流程

1. 收集当日行情、成交、板块、涨停、跌停、资金流、舆情和新闻催化数据；
2. 对底层板块进行成分股映射与数据归一化；
3. 计算底层板块热度分；
4. 计算底层板块延续性分；
5. 识别风险项并计算风险扣分；
6. 计算底层板块主线候选分；
7. 按成分股重叠、核心股共现、舆情关键词、涨停共现和资金同步性进行主线聚合；
8. 计算上层主线分；
9. 计算模型置信度；
10. 生成板块主线排行榜；
11. 生成单主线详情页；
12. 存储模型版本、参数版本、输入数据快照和输出结果；
13. 生成日度复盘报告。

### 5.2 盘中监控流程

1. 按分钟级或5分钟级拉取行情、成交、涨停、触板和舆情数据；
2. 更新板块热度分和风险扣分；
3. 对延续性分进行低频更新，避免盘中噪声过度影响；
4. 监控主线排名变化；
5. 触发异常信号提醒，例如主线突然降温、核心股炸板、资金接力断裂、舆情过热；
6. 在盘中看板展示实时主线状态。

### 5.3 历史回测流程

1. 选择历史区间；
2. 选择模型版本和参数版本；
3. 逐日重放历史数据；
4. 生成每日板块主线排名；
5. 计算次日、3日、5日收益表现；
6. 计算因子IC、Rank IC、命中率、胜率、回撤和拥挤风险；
7. 输出模型评估报告；
8. 支持不同权重组合和风险扣分规则对比。

## 6. 数据需求

### 6.1 数据源类型

| 数据类别 | 主要字段 | 更新频率 | 说明 |
|---|---|---:|---|
| 个股行情 | 开高低收、涨跌幅、成交额、成交量、换手率、流通市值 | 实时/日度 | 核心基础数据 |
| 板块行情 | 板块涨跌幅、成交额、成交量、换手率、成分股数量 | 实时/日度 | 底层板块评分基础 |
| 板块成分 | 板块名称、成分股、权重、分类来源 | 日度/周度 | 需保存历史版本 |
| 涨停数据 | 涨停股、触板股、炸板股、连板高度、封单金额 | 实时/日度 | 情绪与风险识别 |
| 跌停数据 | 跌停股、连续跌停、跌停封单 | 实时/日度 | 风险环境识别 |
| 资金流数据 | 主力净流入、ETF流入、融资余额、北向活跃度 | 日度/分钟级视数据源而定 | 资金结构辅助 |
| 舆情数据 | 股吧热度、雪球热度、搜索热度、新闻提及量、社媒讨论量 | 实时/日度 | 拆分绝对热度和边际变化 |
| 新闻催化 | 政策、产业事件、价格变化、订单、业绩、海外映射 | 实时/日度 | 可半自动打标签 |
| 龙虎榜 | 上榜原因、买卖营业部、净买入金额、席位类型 | 日度 | 定性辅助，不作为高权重因子 |
| 指数环境 | 沪深300、中证全指、万得全A、创业板指、科创50等 | 实时/日度 | 相对强度与市场状态 |
| 交易日历 | 交易日、节假日、特殊交易安排 | 日度 | 调度与回测基础 |

### 6.2 数据质量要求

1. 所有行情数据必须包含交易日、时间戳和数据来源；
2. 板块成分股必须保留历史版本，不得用当前成分回填历史；
3. 涨停数据需区分10cm、20cm、30cm、ST、新股、北交所等不同制度；
4. 舆情数据需进行去重、反垃圾和异常峰值处理；
5. 新闻催化需支持人工校正标签；
6. 缺失数据应标记缺失原因，不能静默填充为0；
7. 所有模型输出必须能追溯到当时可用的数据快照，避免未来函数。

### 6.3 数据延迟要求

| 场景 | 延迟要求 |
|---|---:|
| 盘中行情更新 | 不超过60秒 |
| 涨停/炸板状态更新 | 不超过30秒，视数据源能力调整 |
| 舆情热度更新 | 不超过5分钟 |
| 收盘日度评分 | 收盘后15分钟内生成初版 |
| 完整复盘报告 | 收盘后60分钟内生成正式版 |
| 回测任务 | 支持异步执行，结果可追溯 |

## 7. 板块与主线体系需求

### 7.1 底层板块体系

系统需支持至少以下底层板块类型：

1. 申万一级、二级、三级行业；
2. 中信行业；
3. 同花顺概念；
4. 东方财富概念；
5. 通达信概念；
6. 自定义概念板块；
7. ETF跟踪板块；
8. 指数成分板块。

### 7.2 上层主线体系

上层主线是系统最终输出的主要对象。上层主线可由系统自动聚合，也可由人工维护。

示例：

| 上层主线 | 可包含底层板块 |
|---|---|
| AI硬件/算力基础设施 | CPO、光模块、PCB、服务器、交换机、算力芯片、液冷、铜连接 |
| 国产算力 | 国产GPU、AI芯片、服务器、算力租赁、信创、国产替代 |
| 资源涨价 | 稀土、小金属、铜、铝、锂矿、磷化工、煤炭、油气 |
| 新能源反弹 | 锂电、储能、光伏、固态电池、钠电池、复合集流体 |
| AI应用 | 游戏、传媒、短剧、教育、办公软件、多模态、智能体 |
| 低空经济 | 飞行汽车、eVTOL、无人机、通航、空管系统 |
| 防御红利 | 电力、煤炭、银行、公用事业、高股息 |
| 医药复苏 | CXO、创新药、中药、医疗器械、减肥药 |

### 7.3 主线聚合规则

系统需基于以下维度将底层板块聚合为上层主线：

| 聚合维度 | 说明 |
|---|---|
| 成分股重叠度 | 两个底层板块成分股重叠越高，越可能属于同一主线 |
| 核心股共现 | 多个底层板块由同一批核心股带动，应聚合 |
| 涨停股共现 | 同一批涨停股同时属于多个概念，说明概念关联较强 |
| 舆情关键词相似度 | 新闻和讨论中频繁共现的关键词可辅助聚类 |
| 资金流同步性 | 成交额和涨跌幅同步变化的板块可聚合 |
| 人工配置 | 支持研究员手动修正主线归属 |

### 7.4 主线版本管理

1. 主线定义必须保存版本；
2. 每次主线合并、拆分、重命名需记录操作人、时间和原因；
3. 回测时必须使用历史当时的主线定义版本；
4. 支持查看某条主线历史包含哪些底层板块；
5. 支持某只股票历史上属于哪些主线。

## 8. 评分模型需求

### 8.1 总体评分公式

系统主线分公式为：

```text
主线分 = 热度权重 × 热度分 + 延续性权重 × 延续性分 - 风险扣分
```

默认配置：

```text
主线分 = 0.4 × 热度分 + 0.6 × 延续性分 - 风险扣分
```

根据用户交易周期可配置：

| 交易周期 | 热度权重 | 延续性权重 |
|---|---:|---:|
| 偏短线，1—3天 | 0.5 | 0.5 |
| 中短线，3—7天 | 0.4 | 0.6 |
| 中线，1—4周 | 0.3 | 0.7 |

### 8.2 热度分计算

热度分用于识别当日市场关注度和资金进攻强度，满分100。

| 因子 | 权重 | 说明 |
|---|---:|---|
| 成交活跃度 | 26 | 板块成交额、成交额占比、相对均值放大倍数 |
| 涨停与短线情绪 | 24 | 涨停率、连板高度、封板质量、触板强度 |
| 当日价格强度 | 14 | 当日涨幅、收盘位置、相对大盘强度 |
| 板块广度 | 14 | 上涨家数占比、中位数涨幅、跑赢指数比例 |
| 舆情边际变化率 | 8 | 今日舆情相对过去3日/5日均值变化 |
| 舆情绝对热度 | 4 | 当前讨论量、搜索热度、新闻提及量 |
| 催化强度 | 6 | 政策、涨价、订单、业绩、海外映射等 |
| 容量与可交易性 | 4 | 成交容量、中军承接、ETF可交易性 |
| 合计 | 100 |  |

### 8.3 延续性分计算

延续性分用于评估未来1—3个交易日继续走强的可能性，满分100。

| 因子 | 权重 | 说明 |
|---|---:|---|
| 成交额持续性 | 22 | 连续放量、成交额稳定、无明显衰减 |
| 板块广度持续性 | 20 | 后排扩散、中位数涨幅稳定、上涨家数持续 |
| 核心股结构 | 18 | 龙头、中军、后排结构健康 |
| 涨停质量 | 14 | 封板率高、炸板率低、连板晋级合理 |
| 价格相对强度 | 12 | 3日/5日相对强度、趋势保持 |
| 催化持续性 | 8 | 催化可持续，不只是一次性消息 |
| 舆情边际变化 | 4 | 关注度仍在扩散但未极端过热 |
| 容量与中军承接 | 2 | 有中军股承接大资金 |
| 合计 | 100 |  |

### 8.4 风险扣分计算

风险扣分为0—20，最终从主线分中扣除。

| 风险项 | 扣分区间 | 触发逻辑 |
|---|---:|---|
| 板块连续高潮 | 0—4 | 连续多日大涨、涨停潮、成交极端放大 |
| 炸板率过高 | 0—4 | 触板较多但封板失败比例高 |
| 核心股走弱 | 0—5 | 龙头炸板、中军跳水、核心股尾盘回落 |
| 资金接力断裂 | 0—5 | 昨日领涨股不延续，每天换一批股票涨 |
| 舆情过热 | 0—3 | 热度极高但成交、价格、核心股不再配合 |
| 高位放量滞涨 | 0—4 | 成交放大但涨幅收窄或冲高回落 |
| 后排不跟/广度不足 | 0—3 | 只有少数核心股强，后排无赚钱效应 |
| 监管/异动风险 | 0—3 | 严重异动、监管关注、连续异常波动 |

扣分封顶规则：

```text
风险扣分 = min(20, 各项风险扣分合计)
```

## 9. 关键因子计算需求

### 9.1 成交活跃度

成交活跃度应综合以下指标：

1. 板块成交额排名；
2. 板块成交额占全市场比例；
3. 板块成交额相对5日、20日均值放大倍数；
4. 板块成交额连续放量天数；
5. 板块换手率；
6. 成交集中度。

推荐计算：

```text
成交活跃度 = f(
  成交额分位数,
  成交额占比,
  成交额/20日均值,
  连续放量天数,
  换手率分位数
)
```

### 9.2 涨停与短线情绪

需区分不同涨跌幅制度：

```text
折算涨停数 = 10cm涨停数 + 1.5 × 20cm涨停数 + 2.0 × 30cm涨停数
```

核心指标包括：

1. 涨停家数；
2. 涨停率；
3. 折算涨停数；
4. 触板家数；
5. 炸板率；
6. 封板率；
7. 最高连板高度；
8. 连板家数；
9. 封单金额/流通市值。

### 9.3 板块广度

板块广度用于判断赚钱效应是否扩散。

核心指标包括：

1. 上涨家数占比；
2. 板块中位数涨幅；
3. 跑赢沪深300/中证全指比例；
4. 站上5日均线比例；
5. 站上10日均线比例；
6. 后排股票平均涨幅；
7. 非核心股成交额变化。

### 9.4 价格相对强度

价格相对强度包括：

1. 当日涨幅排名；
2. 3日相对强度；
3. 5日相对强度；
4. 10日相对强度；
5. 收盘位置；
6. 尾盘回落幅度；
7. 是否放量突破近期平台。

### 9.5 核心股结构

系统需识别每条主线的核心股，并判断核心股状态。

核心股类型包括：

| 类型 | 说明 |
|---|---|
| 龙头股 | 涨幅、连板、辨识度领先 |
| 中军股 | 成交额大、市值高、承接大资金 |
| 弹性股 | 小市值、高波动、短线弹性强 |
| 趋势股 | 沿均线趋势上行，不一定涨停 |
| 后排补涨股 | 板块扩散阶段被资金挖掘的低位股票 |

核心股状态评分应关注：

1. 龙头是否继续晋级；
2. 中军是否放量上涨或维持强趋势；
3. 后排是否扩散；
4. 核心股是否炸板；
5. 核心股是否尾盘跳水；
6. 核心股成交是否高位滞涨；
7. 龙头、中军、后排是否同步。

### 9.6 资金接力断裂

资金接力断裂是重要风险因子。

#### 9.6.1 领涨股延续率

```text
领涨延续率 = 昨日领涨股中今日继续跑赢板块中位数的数量 / 昨日领涨股数量
```

判断标准：

| 领涨延续率 | 信号 |
|---:|---|
| > 60% | 接力健康 |
| 40%—60% | 正常分化 |
| 20%—40% | 接力变弱 |
| < 20% | 明显断裂 |

#### 9.6.2 涨停重合率

```text
涨停重合率 = 今日涨停且昨日也强势的股票数 / 今日涨停股票数
```

判断标准：

| 涨停重合率 | 信号 |
|---:|---|
| 30%—50% | 接力良好 |
| 15%—30% | 轮动正常 |
| < 15% | 接力偏弱 |
| 接近0且涨停仍多 | 高度警惕，可能是补涨末期 |

#### 9.6.3 核心股偏离度

```text
核心股偏离度 = 板块中位数涨幅 - 核心股平均涨幅
```

如果核心股显著弱于板块中位数，说明资金可能从核心股撤离，转向后排补涨。

资金接力断裂扣分建议：

| 条件 | 扣分 |
|---|---:|
| 领涨延续率低于40% | -1 到 -2 |
| 涨停重合率极低且每天换票 | -1 到 -2 |
| 核心股明显弱于后排 | -2 到 -3 |
| 三项同时触发 | -4 到 -5 |

### 9.7 舆情因子

舆情需拆成三个部分。

#### 9.7.1 舆情边际变化

```text
舆情变化率 = 今日舆情热度 / 过去3日或5日平均舆情热度 - 1
```

该指标用于捕捉冷门板块突然升温、新催化出现、市场开始发现新方向。

#### 9.7.2 舆情绝对热度

舆情绝对热度低权重加分，用于确认市场关注度。

数据来源可包括：

1. 股票社区讨论量；
2. 财经媒体提及量；
3. 搜索指数；
4. 新闻推送频率；
5. 热榜排名；
6. 关键词出现频次。

#### 9.7.3 舆情过热扣分

以下情况触发舆情过热扣分：

1. 连续多日热度排名第一；
2. 讨论量继续上升但板块价格不创新高；
3. 成交额放大但涨幅收窄；
4. 核心股走弱但后排补涨；
5. 社区和财经媒体集中刷屏；
6. 热度高但资金净流入下降。

### 9.8 催化强度

催化强度可分为四档：

| 等级 | 分值 | 示例 |
|---|---:|---|
| S级 | 8—10 | 国家级政策、行业周期反转、全球大厂订单、产品价格持续上涨 |
| A级 | 6—8 | 行业会议、业绩高增、供需边际改善、海外映射 |
| B级 | 3—5 | 个股公告、短期消息、局部涨价、主题活动 |
| C级 | 0—2 | 传闻、小作文、无明确来源信息 |

系统需支持人工对催化等级进行修正。

## 10. 因子动态调整需求

### 10.1 基本原则

系统不应根据主观市场环境标签大幅切换权重，例如“牛市权重”“熊市权重”“题材行情权重”。

系统应基于因子近期有效性进行小幅动态修正：

```text
最终因子权重 = 0.85 × 基础权重 + 0.15 × 动态权重
```

### 10.2 双窗口确认机制

系统需同时计算5日和20日因子有效性。

| 5日有效性 | 20日有效性 | 调整动作 |
|---|---|---|
| 上升 | 上升 | 小幅上调 |
| 下降 | 下降 | 小幅下调 |
| 上升 | 下降 | 不调整 |
| 下降 | 上升 | 不调整 |
| 不显著 | 任意 | 不调整 |

### 10.3 指数衰减备选机制

可配置指数衰减计算：

| 时间窗口 | 权重 |
|---|---:|
| 最近1—5日 | 40% |
| 最近6—10日 | 30% |
| 最近11—20日 | 30% |

```text
因子有效性 = 0.4 × IC_5日 + 0.3 × IC_6至10日 + 0.3 × IC_11至20日
```

### 10.4 调整边界

| 限制项 | 要求 |
|---|---:|
| 单因子单日权重变化 | 不超过1—2个百分点 |
| 单因子最大上调 | 不超过基础权重的25% |
| 单因子最大下调 | 不超过基础权重的25% |
| 成交活跃度权重下限 | 不低于20% |
| 板块广度权重下限 | 不低于15% |
| 舆情绝对热度权重上限 | 不高于5% |
| 涨停情绪权重上限 | 不高于28% |

## 11. 置信度计算需求

### 11.1 置信度定义

置信度用于说明模型当日排序的可靠性，分为高、中、低三档。

| 置信度 | 含义 |
|---|---|
| 高 | 主线清晰，成交和情绪共振，排名分差明显，风险可控 |
| 中 | 有主线但存在分歧，部分指标矛盾，需继续验证 |
| 低 | 板块轮动混乱、成交不足、风险普遍偏高，排序参考价值有限 |

### 11.2 置信度计算维度

| 维度 | 权重 | 说明 |
|---|---:|---|
| 流动性分 | 30% | 全市场成交额/20日均成交额、成交是否过低 |
| 主线分差 | 25% | 排名前3主线分差是否明显 |
| 风险稳定性 | 20% | 前10主线风险扣分均值是否过高 |
| 市场广度 | 15% | 上涨家数占比、市场中位数涨幅 |
| 主线一致性 | 10% | 底层分支是否能聚合成清晰主线 |

推荐公式：

```text
置信度分 = 0.30 × 流动性分 + 0.25 × 主线分差分 + 0.20 × 风险稳定性分 + 0.15 × 市场广度分 + 0.10 × 主线一致性分
```

### 11.3 置信度分档

| 置信度分 | 档位 |
|---:|---|
| >= 75 | 高 |
| 55—75 | 中 |
| < 55 | 低 |

### 11.4 置信度说明输出

系统必须输出置信度原因，例如：

```text
今日模型置信度：中。
原因：全市场成交额充足，但排名前3主线分差不大；AI硬件高位分歧，资源线升温但尚未连续确认；短线涨停仍活跃，但炸板和资金接力断裂风险上升。
```

## 19. 风险与约束

### 19.1 数据风险

1. 不同数据源板块定义不一致；
2. 舆情数据可能存在刷量和噪声；
3. 涨停统计口径可能因ST、新股、北交所不同而不一致；
4. 板块成分历史缺失可能导致回测偏差；
5. 新闻催化自动识别存在误判。

### 19.2 模型风险

1. 评分模型可能过拟合历史行情；
2. 因子权重过度调整会降低稳定性；
3. 快速轮动行情下延续性分有效性下降；
4. 高成交主线容易被过度拥挤信号误伤；
5. 板块聚合错误会导致主线输出失真。

### 19.3 产品风险

1. 用户可能误把主线排名理解为投资建议；
2. 低置信度提示不明显会导致误用；
3. 过多指标可能造成页面复杂；
4. 人工维护主线体系会带来运营成本。

### 19.4 风险缓解措施

1. 明确标注“研究辅助，不构成投资建议”；
2. 所有模型输出附带置信度；
3. 低置信度时弱化排序视觉权重；
4. 提供分数解释和风险明细；
5. 保持模型版本和参数版本可回溯；
6. 定期回测模型有效性；
7. 支持人工修正主线定义。
